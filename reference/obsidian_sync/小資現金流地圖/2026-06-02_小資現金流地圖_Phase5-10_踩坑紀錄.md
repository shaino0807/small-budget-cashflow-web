# 2026-06-02 小資現金流地圖 Phase 5-10 踩坑紀錄

## 1. PowerShell 預設編碼造成誤判

現象：

- 用一般 `Get-Content` 讀 Swift 檔時，繁中內容看起來像亂碼。
- 一開始誤以為 `apply_patch` 沒有成功寫入。

原因：

- Windows PowerShell 預設輸出編碼與 UTF-8 Swift 檔不一致。

處理：

- 讀 Swift 檔一律使用 `Get-Content -Encoding UTF8`。
- 靜態掃描也改用 UTF-8。

教訓：

- 在 Windows 編輯 Swift + 繁中文案時，檔案內容驗證要指定 UTF-8，否則會把正常檔案誤判成壞檔。

## 2. Xcode project 是明確列檔模式

現象：

- 新增 Swift 檔後，如果不改 `project.pbxproj`，Xcode 不會編譯新檔。

影響：

- `PaidReportView.swift`
- `PurchaseManager.swift`
- `UpgradeView.swift`
- `UpgradePlanCard.swift`
- `SimulationView.swift`
- `CashflowCalendarView.swift`
- `PDFReportService.swift`
- `ReportComponents.swift`

處理：

- 每新增一個 Swift 檔，都同步加入：
  - `PBXBuildFile`
  - `PBXFileReference`
  - 對應 group children
  - `PBXSourcesBuildPhase`

教訓：

- 這個專案不是 Xcode 新式 file system synchronized group，新增檔案後一定要更新 project file。

## 3. `UpgradeView` 一度留下重複舊內容

現象：

- 局部 patch 後，`UpgradeView.swift` 尾端殘留舊版本內容。
- 這會造成 Swift 語法錯誤。

處理：

- 讀回整檔確認。
- 直接整檔重寫 `UpgradeView.swift`。

教訓：

- UI 重構如果跨很多段落，局部 patch 後要讀回整檔，避免尾段殘留。

## 4. PDF skeleton 要選擇最小可行做法

現象：

- Phase 9 可以只做空方法，但如果完全空方法，後面 UI 很難驗證按鈕是否真的接上。

處理：

- 建立 `PDFReportService.exportPDF(report:) throws -> URL`
- 使用 `UIGraphicsPDFRenderer` 產生 temporary directory 裡的簡易 PDF。
- 內容先用文字區塊，未來再換正式排版。

教訓：

- skeleton 最好保留真實輸入輸出形狀，哪怕內容簡單，也比空方法更容易往後擴充。

## 5. PDF service 引入 UIKit

現象：

- `PDFReportService` 使用 `UIGraphicsPDFRenderer`，需要 UIKit。

風險：

- 目前是 iOS app，合理。
- 但如果未來要做 macOS target 或 Swift Package，就要改成平台條件編譯或換 PDFKit / CoreGraphics。

處理：

- 目前保留 `import UIKit`。
- 記錄為未來跨平台時需處理的點。

## 6. Swift Charts 只能在可用平台上編譯驗證

現象：

- `SimulationView` 使用 `import Charts`。

風險：

- 需要 iOS 16+。
- 專案目前 deployment target 是 iOS 17.0，理論上可用。

限制：

- Windows 環境沒有 `xcodebuild`，無法實際編譯確認。

處理：

- 已做靜態檢查。
- 最終需在 macOS + Xcode build。

## 7. 免費版與付費版文案邏輯要分開

現象：

- 一開始 `FinancialHealthEngine.nextBestActions` 是免費版鎖定提示用文案。
- 若直接放到付費報告，會出現「完整版會...」這種不適合付費頁的文字。

處理：

- `PaidReportView` 內建立 `personalizedActionItems`，用分數拆解產生真正的個人化建議。

教訓：

- 免費版文案是 teaser；付費版文案要給可執行方向。兩者不應共用同一組內容。

## 8. 免責聲明必須集中管理

現象：

- App 畫面與 PDF 都需要免責聲明。
- 如果每處手寫，未來容易不一致。

處理：

- 建立 `AppDisclaimer.text`。
- 免費報告、付費報告、付費牆、PDF service 都引用同一份文字。

教訓：

- 法務 / 風險提示類文字應集中管理，避免不同畫面版本不一致。

## 9. Google Drive Vault 寫入需要授權

現象：

- `G:\我的雲端硬碟` 超出目前工作區 sandbox。
- 一開始讀取被拒。

處理：

- 使用 escalated command 檢查雲端資料夾。
- 確認 `G:\我的雲端硬碟\oB 與 Obsidian 同步更新` 已有 `.obsidian`，不需要新建 Vault。

教訓：

- 雲端同步資料夾屬於工作區外部路徑，所有寫入都要明確授權。
