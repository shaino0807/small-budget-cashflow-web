# 小資現金流地圖 踩坑紀錄

日期：2026-06-02

## 1. Windows PowerShell 顯示中文亂碼，但檔案本身不一定壞掉

### 現象

用 PowerShell `Get-Content` 看 Swift 檔時，中文字串看起來像亂碼，甚至像是字串引號壞掉。

### 判斷方式

不能只看 PowerShell 輸出。後來用 Python 以 UTF-8 讀檔，再用 `ascii()` 檢查內容，確認實際檔案裡是正常 Unicode。

### 結論

之後遇到中文亂碼時，先判斷是「終端顯示問題」還是「檔案內容真的被污染」。不要急著重寫檔案。

## 2. XcodeBuildMCP 在目前 Windows 環境無法真正 build iOS App

### 現象

執行 simulator build 時回傳：

```text
spawn /bin/sh ENOENT
```

或列 scheme 時出現 xcodebuild 不存在。

### 原因

目前環境是 Windows，沒有 macOS shell 與 Xcode runtime。XcodeBuildMCP 工具可被呼叫，但無法在這台環境完成真正的 iOS 編譯。

### 結論

目前只能做靜態檢查與 project source reference 檢查。真正編譯需要移到 macOS / Xcode 環境。

## 3. 手動維護 project.pbxproj 容易漏加 Sources

### 現象

新增 Swift 檔後，如果只把檔案放進資料夾，Xcode target 不一定會編譯它。

### 解法

每次新增 Swift 檔，都要確認 project.pbxproj 有三個地方：

- PBXBuildFile
- PBXFileReference
- PBXGroup
- PBXSourcesBuildPhase

### 結論

在沒有 Xcode GUI 或 XcodeGen 的情況下，手動維護 project.pbxproj 是可行的，但每次都要做 source reference 檢查。

## 4. Decimal 輸入需要獨立 helper

### 現象

SwiftUI 表單中的金額欄位用 String 綁定比較自然，但核心 model 使用 Decimal。

### 解法

新增 Decimal / String helper：

- Decimal.inputText
- String.decimalValue

### 結論

不要在每個 View 裡重複轉型邏輯。金額輸入集中處理，比較容易維持表單驗證一致。

## 5. Phase 4 檢查時不要只看「畫面有沒有做」

### 檢查清單

Phase 4 不只是新增 View，還要確認：

- ContentView 是否接到 OnboardingView
- OnboardingView 是否用 NavigationStack
- FinancialInputView 是否有驗證與 DecimalPad
- ETFInputView 是否能新增與刪除 ETF
- 產生報告是否真的呼叫 generateFreeReport()
- 是否導向 FreeReportView
- 新檔案是否加入 Xcode project Sources

### 結論

這次 Phase 4 功能面已完成。剩下主要風險是尚未在 macOS / Xcode 進行真實編譯。

## 6. PowerShell 版本差異：New-Item 不一定支援 -LiteralPath

### 現象

同步到 Google Drive 時，原本想用：

```powershell
New-Item -ItemType Directory -Force -LiteralPath $vaultProject
```

但目前 PowerShell 回傳：

```text
A parameter cannot be found that matches parameter name 'LiteralPath'.
```

接著 Copy-Item 把原本要當作資料夾的路徑誤當成檔案名稱，產生了一個 `小資現金流地圖` 檔案。

### 修正方式

沒有刪除該檔案，而是先保留並改名成：

```text
小資現金流地圖_同步誤建檔案備份.md
```

再用 `New-Item -Path` 建立真正的資料夾，重新複製收工紀錄與踩坑紀錄。

### 結論

在 Windows PowerShell 上建立資料夾時，若不確定版本支援狀況，優先使用 `-Path`，並在 Copy-Item 前確認目標是 Container，不要只假設路徑會自動變成資料夾。
