## 踩坑補充 2026-06-10

- `update-price-series.js` 原本硬寫死 `20260609`，導致隔天仍查舊日期。已改成用 `Asia/Taipei` 產生當日查詢日。注意：查詢日是 20260610 不代表官方一定已發布 6/10 行情；資料列仍以官方回傳日期為準，不能自行補假資料。
- `update-issuer-official-data.js` 每跑一次都會重複塞入同一句 limitation，造成每日 Actions 可能產生無意義 diff。已改成先清掉同類 limitation，再寫入單一最新描述。
- GitHub CLI 原本的 `shaino0807` token 失效，`gh auth login -w` 在 Codex shell 中沒有穩定顯示 device code。最後改用 GitHub OAuth device flow 取得 code，使用者手動授權後再用臨時 token 建 repo、push、啟用 Pages。
- 第一次 GitHub Actions 失敗是因為 push workflow 時 Pages 還沒啟用，`actions/configure-pages` 查不到 Pages site。啟用 Pages 後 rerun 即成功。
- GitHub Actions 會自動 commit 更新後的 `webapp/data/etf-database.json`，因此本地後續 push 可能被拒絕 `fetch first`。處理方式：先 `pull --rebase`，保留本地未提交變更，再 push。
- GitHub Pages 是靜態環境，沒有 `/api/database-status` 或 `/api/update-database`。公開頁若沿用本機 API 呼叫會造成 404 console error。已在 `location.hostname.endsWith("github.io")` 時跳過伺服器更新 API。
- GitHub Pages 會向站台根路徑要求 favicon；若只靠 `icon.svg` 但沒宣告 `<link rel="icon">`，會產生 favicon 404。已在 `index.html` 補上 SVG favicon link。
- Service Worker 會讓遠端頁測試誤讀舊的 `app.js`。每次修公開前端資產後要 bump `sw.js` cache name，測試時也要清掉舊 service worker/cache 再判斷是否真的修好。
- 遠端 smoke test 需同時看 HTML 200、JSON 200、console errors、Network 4xx、手機 `bodyOverflow`、完整報告解鎖後的表格列數；只看首頁 200 不代表部署成功。
