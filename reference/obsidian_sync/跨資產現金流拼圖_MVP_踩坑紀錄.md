# 跨資產現金流拼圖 MVP 踩坑紀錄

日期：2026-06-01

## 1. 原始文件與前端檔案有編碼損壞

現象：

- `LINE_AI_現金流教練_MVP_Workflow.md` 內容出現大量亂碼。
- 既有 `index.html`、`styles.css`、`app.js` 也有中文亂碼。
- 原 `app.js` 內有字串未閉合、語法損壞等問題。

處理：

- 沒有覆蓋舊 Workflow，避免破壞原始資料。
- 新增乾淨版 `跨資產現金流拼圖視覺化_App_MVP_Workflow.md`。
- 重新建立 `index.html`、`styles.css`、`app.js`，統一使用 UTF-8。

後續注意：

- Markdown 與前端檔案都應固定 UTF-8。
- 若從其他工具匯出中文內容，先用編輯器確認編碼再提交。

## 2. 先做半自動模擬，不要過早做完整金融系統

現象：

- 需求包含現金流、ETF、保險、貸款、信用卡回饋、配息日曆、退休試算、產業集中度，容易一開始就膨脹成大型系統。

處理：

- 第一版只做靜態互動原型。
- 不串銀行、券商、投信 API。
- 不做下單、不做自動投資建議。
- 用假資料模擬使用者體驗與產品價值。

後續注意：

- 先驗證「用戶是否看得懂一張圖」。
- API、登入、資料庫、同步機制都應等 MVP 驗證後再加入。

## 3. 金融 AI 輸出必須保留確認

風險：

- AI 容易把模擬結果講得像確定建議。
- ETF 高配息、槓桿、反向、保險條款都可能被誤解。

處理：

- UI 中把 AI 輸出設計為「下一步候選動作」。
- 每個候選動作都保留用戶確認。
- 文案避免「必買、必賣、保證獲利、穩賺」。

後續注意：

- 未來接 OpenAI API 時，輸出必須用 JSON schema 驗證。
- 風險提醒應有規則層，不只依賴 LLM。

## 4. ETF 資料更新是獨立問題

現象：

- 配息日曆與 ETF 成分股會變動。
- MVP 不能假設資料永遠正確。

處理：

- 原型顯示「最後更新時間」。
- 配息日曆先做模擬更新按鈕。
- Workflow 中保留 CSV 匯入與人工審核流程。

後續注意：

- 第二階段需要明確資料來源：交易所、投信官網、資料供應商或人工 CSV。
- 若來源衝突，應標記 `needs_review`，不要自動覆蓋。

## 5. 瀏覽器自動化驗證失敗

現象：

- 嘗試用 Node REPL + Playwright 開啟本地 `index.html` 做自動檢查。
- 工具回報 `node_repl kernel exited unexpectedly` 與 `windows sandbox failed: spawn setup refresh`。

處理：

- 改做靜態檢查。
- 已執行 `node --check app.js`，語法通過。
- 也檢查 `index.html` 開頭內容，確認 UTF-8 中文正常。

後續注意：

- 若要正式驗證 UI，建議在本機瀏覽器手動開啟 `index.html`。
- 或改用可穩定啟動的本地 dev server + Browser plugin。

## 6. G 槽雲端 Vault 需要額外權限

現象：

- 直接讀取 `G:\我的雲端硬碟\oB 與 Obsidian` 被沙盒拒絕。

處理：

- 使用 escalated filesystem 權限檢查。
- 確認該資料夾已存在 `.obsidian`，所以它已經是 Obsidian Vault。
- 不需要新建 Vault，只需要同步到 Vault 內的專案資料夾。

後續注意：

- 對雲端硬碟寫入、建立資料夾、複製檔案，都需要明確權限。
- 同步前要避免刪除或覆蓋未知檔案。
